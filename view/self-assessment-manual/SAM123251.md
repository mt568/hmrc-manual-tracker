When dealing with the LDC RETURNS - LOGGING FAILURES - NO RECORD FOUND paper report follow steps 1 - 7 below. Note: If the list is received in a Banking Operations office, the return will have been sent on and can be ignored.

For details of how to access the SA functions, select ‘Index Of Functions’ on the left of the screen.

1. Use function LOG RETURN to log the return correctly where the Unique Taxpayer Reference (UTR) was miskeyed

2. Use the Technical Support System (TSS) to enter the return details

3. In Trust and Estate return cases, on receipt of the ACTION REQUIRED FOLLOWING DATE CAPTURE printout

* Take the action set out in subject ‘Calculating trust liability’ (SAM123220) (including the use of function CREATE RETURN CHARGE)

In Tax Return for Trustees of Registered Pension Schemes cases, on receipt of the ACTION REQUIRED FOLLOWING DATE CAPTURE printout

* Take the action set out in subject ‘Processing the Pension Scheme Tax Return’ (SAM123330)

4. Delete the entry from the list

### Where the Organisation Unit Identifier (OUID) used in LOG RETURN is invalid

5. Ask your Local Administrator to correct the OUID using function LDC ADMIN 2000

6. Use function LOG RETURN to log the return using the correct OUID

7. Delete the entry from the list
